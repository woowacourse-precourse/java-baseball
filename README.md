# 미션 - 숫자 야구

## 📌 기능 목록

### 📍 Function List

- 게임 콘솔은 게임을 시작한다.
- 게임 콘솔은 단일 게임을 시작한다.
- 게임 콘솔은 단일 게임 종료 후 새로운 게임을 시작할지 물어본다.
- 타자(컴퓨터)는 임의의 수를 생성한다.
- 투수(사용자)는 3자리 수를 입력하여 알려준다.
- 투수(사용자)는 새로운 게임을 시작할지 입력하여 알려준다.
- 심판은 투수의 숫자 투구가 보크(잘못된 입력)인지 검증한다.
- 심판은 투수와 타자의 숫자를 비교하여 결과를 반환한다.
- 심판은 새로운 게임에 대한 투수의 의사가 잘못된 입력인지 검증한다.
- 진행자는 게임 시작 전 투수에게 안내문을 보여준다.
- 진행자는 심판이 반환한 결과를 투수에게 힌트와 함께 알려준다.
- 진행자는 게임 종료 안내문을 보여준다.

### 📍 시간 순서 리스트

1. 숫자 야구 게임 콘솔은 게임을 시작한다.
2. 진행자는 게임 시작 안내문을 보여준다.
3. 콘솔은 하나의 야구 게임을 시작한다.
4. 타자(컴퓨터)는 임의의 수를 선택하여 콘솔에게 알려준다.
5. 투수(사용자)는 수를 예측하여 콘솔에게 알려준다.
6. 심판은 투수가 예측한 수를 검증한다. (잘못된 입력이면 15번으로 넘어간다.)
    1. 수의 크기를 검증한다.
    2. 수의 숫자들을 검증한다.
7. 심판은 투수가 예측한 수에 대한 결과를 콘솔에게 알려준다.
    1. 심판은 임의의 수와 예측한 수를 통해 결과를 가공한다.
        1. 심판은 임의의 수와 예측한 수를 비교한다.
        2. 심판은 예측한 수의 각 자리수가 볼인지 스트라이크인지 구별한다. (둘 다 아닐 수도 있다.)
8. 진행자는 결과값을 바탕으로 플레이어에게 결과를 보여준다.
9. 임의의 수와 예측한 수가 일치하지 않으면 5번으로 돌아간다.
10. 콘솔은 투수에게 새로운 게임을 시작할 것인지 물어본다.
11. 투수(사용자)는 새로운 게임을 시작할거면 1을 아니면 2를 입력한다.
12. 심판은 투수의 재시작 선택값을 검증한다. (잘못된 입력이면 15번으로 넘어간다.)
13. 투수(사용자)가 새로운 게임을 시작하길 원하면 3번으로 돌아간다.
14. 진행자는 게임 종료 안내문을 보여준다.
15. 게임 종료

### 📍 Class 기능 리스트

- BaseballGameConsole(게임 콘솔) : 야구 게임을 진행하는 클래스
  - 게임을 시작한다.
  - 단일 야구 게임을 시작한다.
  - 투수에게 새로운 게임을 시작할 것인지 물어본다.
- Batter(컴퓨터 타자) : 임의의 수를 생성하는 클래스
  - 임의의 수를 선택한다.
- Pitcher(사용자 투수) : 게임에 문제를 맞추는 클래스
  - 임의의 수를 예측한다.
  - 새로운 게임을 시작할 것인지 선택한다.
- Referee(심판) : 사용자 입력값 검증 및 결과 분석 클래스
  - 투수의 예측 입력값을 검증한다.
  - 투수가 예측한 수와 임의의 수를 비교한다.
  - Replay 에 대한 투수의 입력(1 or 2)을 검증한다.
- 진행자(Emcee)
  - 게임 시작 안내문을 보여준다.
  - 투수의 예측에 대한 결과 메세지를 출력한다.
  - 게임 종료 안내문을 보여준다.

## 🚀 기능 요구 사항

기본적으로 1부터 9까지 서로 다른 수로 이루어진 3자리의 수를 맞추는 게임이다.

- 같은 수가 같은 자리에 있으면 스트라이크, 다른 자리에 있으면 볼, 같은 수가 전혀 없으면 낫싱이란 힌트를 얻고, 그 힌트를 이용해서 먼저 상대방(컴퓨터)의 수를 맞추면 승리한다.
    - 예) 상대방(컴퓨터)의 수가 425일 때
        - 123을 제시한 경우 : 1스트라이크
        - 456을 제시한 경우 : 1볼 1스트라이크
        - 789를 제시한 경우 : 낫싱
- 위 숫자 야구 게임에서 상대방의 역할을 컴퓨터가 한다. 컴퓨터는 1에서 9까지 서로 다른 임의의 수 3개를 선택한다. 게임 플레이어는 컴퓨터가 생각하고 있는 서로 다른 3개의 숫자를 입력하고, 컴퓨터는 입력한 숫자에 대한
  결과를 출력한다.
- 이 같은 과정을 반복해 컴퓨터가 선택한 3개의 숫자를 모두 맞히면 게임이 종료된다.
- 게임을 종료한 후 게임을 다시 시작하거나 완전히 종료할 수 있다.
- 사용자가 잘못된 값을 입력할 경우 `IllegalArgumentException`을 발생시킨 후 애플리케이션은 종료되어야 한다.

### 입출력 요구 사항

#### 입력

- 서로 다른 3자리의 수
- 게임이 끝난 경우 재시작/종료를 구분하는 1과 2 중 하나의 수

#### 출력

- 입력한 수에 대한 결과를 볼, 스트라이크 개수로 표시

```
1볼 1스트라이크
```

- 하나도 없는 경우

```
낫싱
```

- 3개의 숫자를 모두 맞힐 경우

```
3스트라이크
3개의 숫자를 모두 맞히셨습니다! 게임 종료
```

- 게임 시작 문구 출력

```
숫자 야구 게임을 시작합니다.
``` 

#### 실행 결과 예시

```

숫자를 입력해주세요 : 123
```

## 🎯 프로그래밍 요구 사항

- JDK 11 버전에서 실행 가능해야 한다. **JDK 11에서 정상적으로 동작하지 않을 경우 0점 처리한다.**
- 프로그램 실행의 시작점은 `Application`의 `main()`이다.
- `build.gradle` 파일을 변경할 수 없고, 외부 라이브러리를 사용하지 않는다.
- [Java 코드 컨벤션](https://github.com/woowacourse/woowacourse-docs/tree/master/styleguide/java) 가이드를 준수하며 프로그래밍한다.
- 프로그램 종료 시 `System.exit()`를 호출하지 않는다.
- 프로그램 구현이 완료되면 `ApplicationTest`의 모든 테스트가 성공해야 한다. **테스트가 실패할 경우 0점 처리한다.**
- 프로그래밍 요구 사항에서 달리 명시하지 않는 한 파일, 패키지 이름을 수정하거나 이동하지 않는다.

### 추가된 요구 사항

- indent(인덴트, 들여쓰기) depth를 3이 넘지 않도록 구현한다. 2까지만 허용한다.
    - 예를 들어 while문 안에 if문이 있으면 들여쓰기는 2이다.
    - 힌트: indent(인덴트, 들여쓰기) depth를 줄이는 좋은 방법은 함수(또는 메서드)를 분리하면 된다.
- 3항 연산자를 쓰지 않는다.
- 함수(또는 메서드)가 한 가지 일만 하도록 최대한 작게 만들어라.
- JUnit 5와 AssertJ를 이용하여 본인이 정리한 기능 목록이 정상 동작함을 테스트 코드로 확인한다.
    - 테스트 도구 사용법이 익숙하지 않다면 `test/java/study`를 참고하여 학습한 후 테스트를 구현한다.

### 라이브러리

- `camp.nextstep.edu.missionutils`에서 제공하는 `Randoms` 및 `Console` API를 사용하여 구현해야 한다.
    - Random 값 추출은 `camp.nextstep.edu.missionutils.Randoms`의 `pickNumberInRange()`를 활용한다.
    - 사용자가 입력하는 값은 `camp.nextstep.edu.missionutils.Console`의 `readLine()`을 활용한다.

#### 사용 예시

```java
List<Integer> batter = new ArrayList<>();
while (batter.size() < 3) {
    int randomNumber = Randoms.pickNumberInRange(1, 9);
    if (!batter.contains(randomNumber)) {
        batter.add(randomNumber);
    }
}
```

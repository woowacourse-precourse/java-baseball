## 🎯 전체 흐름

1. 게임을 시작하고, 컴퓨터 숫자로 1 ~ 9 서로 다른 3가지 숫자 랜덤 생성
2. 사용자의 숫자 입력
3. 입력받은 숫자와 컴퓨터 숫자 비교 후, 결과 출력
    - 같은 수가 같은 자리에 있으면 스트라이크
    - 같은 수가 다른 자리에 있으면 볼
    - 같은 수가 전혀 없으면 낫싱
4. 3 스트라이크인 경우, 게임 종료
    - 아닌 경우, 2 ~ 4 과정 반복
5. 게임이 종료 된 경우, 게임의 추가 진행 여부 확인
   - 입력으로 1이 들어오면, 재시작
   - 2가 들어오면, 완전 종료

## 🪜 구조

- **domain**
  - `Computer` : 컴퓨터가 랜덤 생성한 숫자 저장
  - `Game` : 볼, 스트라이크 상황 저장
  - `User` : 사용자가 입력한 숫자 저장
- **service**
  - `GameService` : 게임 진행에 필요한 기능들 구현
- **utils**
  - `CheckException` : 예외 처리
